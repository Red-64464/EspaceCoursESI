Les pointeurs — qu'est-ce que c'est ?

Imagine que ta mémoire est un immeuble. Chaque appartement a un numéro (l'adresse) et contient une valeur.

Un pointeur = une variable qui contient l'adresse d'une autre variable (pas la valeur, juste l'adresse !).

Les références — un surnom !

Une référence c'est comme un surnom pour une variable. Ce n'est pas une copie, c'est la même chose avec un autre nom.

Pointeur vs Référence

Pointeur contient une adresse
int* p = &a;
*p = 10; // déréférence
p->methode(); // objet

Peut pointer ailleurs. Peut être null.

Référence alias direct
int& ref = a;
ref = 10; // comme a = 10
ref.methode(); // objet

Toujours la même variable. Ne peut pas être null.

a = 5
↕ même chose
ref = 5
Les deux variables pointent vers la même valeur.

Memory Leak — la fuite mémoire

Si tu crées des objets avec new mais que tu oublies delete, la mémoire se remplit sans jamais se vider. C'est la fuite mémoire.

Image : louer des boîtes sans jamais les rendre. La RAM monte, monte, monte...

Simulation de la boucle infinie

void triggerMemoryLeak() {
  while (true) {
    Song* song = new Song(...);
    // jamais de delete !
  }
}

C'est quoi un Buffer ?

Imagine une file d'attente devant un guichet. Les gens arrivent, font la queue, et passent un par un. Le buffer c'est pareil : c'est une zone mémoire temporaire qui stocke des données en attendant qu'elles soient traitées.

En C++, un buffer c'est simplement un tableau (array) qui garde des valeurs en mémoire avant qu'on les utilise.

Clique sur une brique pour comprendre le concept.

L'AudioBuffer — un buffer pour le son

Un AudioBuffer c'est un buffer spécialisé : il contient des échantillons audio — des nombres entre -1.0 et +1.0 qui représentent le son.

Chaque nombre = la position du haut-parleur à un instant donné. Mis bout à bout, ça fait un son !

// Un AudioBuffer, c'est juste un tableau de float
float audioBuffer[8] = {
  0.0, 0.5, 0.9, 0.5,
  0.0, -0.5, -0.9, -0.5
};
Visualisation : clique sur chaque case pour voir sa valeur
Chaque case du tableau = un échantillon audio. Clique sur une case !

Du buffer au son — le sinus

Si on remplit le buffer avec la formule sin(), on obtient une onde sinusoïdale — le son le plus pur qui existe (comme un sifflement).

La formule : buffer[i] = sin(2π × fréquence × i / sampleRate)

Échantillons 16
Plus il y a d'échantillons, plus la courbe est précise. 16 points suffisent pour voir la forme d'onde.

La classe AudioPlayer

L'AudioPlayer prend une Song, accède à son tableau de valeurs audio, applique le volume, et affiche les résultats.

La formule : audioOut = volume × audioIn — puis on limite entre -1 et 1.
