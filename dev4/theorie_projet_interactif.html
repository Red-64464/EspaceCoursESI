C'est quoi un son ?

Le son = une vibration dans l'air. L'air bouge, ça crée une onde de pression qui voyage jusqu'à ton oreille → tu entends.

C'est quoi un sample (échantillon) ?

Un sample = une mesure du son à un instant précis. Comme une photo du son.

ANALOGIE VIDÉO 🎥
Vidéo = plein d'images
Son = plein de nombres

Fréquence d'échantillonnage — c'est quoi ?

C'est combien de samples on prélève par seconde. Unité : Hz (Hertz).

⏱ Exemple : 44100 Hz = 44 100 samples par seconde.
En 1 seconde, l'ordi mesure l'onde 44 100 fois.

C'est quoi un fichier .WAV ?

Un .wav = une boîte remplie de nombres (samples). Le fichier stocke les données audio sans compression.

WAV ✅
✓ Qualité parfaite
✓ Facile à lire en C++
✓ Non compressé
✗ Fichier lourd
MP3
✓ Fichier léger
✗ Perd de la qualité
✗ Plus complexe à décoder
✗ Non utilisé dans le projet

C'est quoi une boîte à rythmes ?

C'est une machine qui joue des sons automatiquement et en boucle. Un séquenceur gère une grille : lignes = instruments, colonnes = pas de temps.

Temps → [ 1 ][ 2 ][ 3 ][ 4 ][ 5 ][ 6 ][ 7 ][ 8 ]...
Kick → [ X ][ ][ X ][ ][ X ][ ][ ][ ]
Snare → [ ][ X ][ ][ X ][ ][ X ][ ][ ]
Hi-Hat → [ X ][ X ][ X ][ X ][ X ][ X ][ X ][ X ]

Un curseur avance case par case → case active ✅ = son joué → arrivée à la fin → retour au début 🔁

Calcul : BPM → durée d'un step

durée_beat = 60 / BPM // en secondes
durée_step = durée_beat / 4 // 4 subdivisions par beat

// À 120 BPM :
durée_beat = 60 / 120 = 0.5 s
durée_step = 0.5 / 4 = 0.125 s = 125 ms

Résumé — ce qu'il faut absolument retenir

Séquenceur — décide quand jouer (grille 4×16, BPM)
Audio Player ×4 — lit son fichier WAV (kick, snare, hihat, clap)
Delay ×4 — ajoute un écho à la piste (delay_time + delay_mix)
Mixer — additionne les 4 sons en un seul signal stéréo
Audio Out — envoie vers les haut-parleurs / casque 🔊
